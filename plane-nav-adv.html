Plane Navigation Advanced

Pick a coordinate. Open the plane. Swivel the view.

This describes the “advanced plane traversal” layer: you can jump to any image at a known (x,y,z) coordinate, then build a navigable slice around it using offset distances and a rotation dial. The resulting plane view can be saved as a reproducible snapshot so a screenshot isn’t just pretty, it’s a map back to the same cluster.

Core metaphor: like a door on a swivel. You stand on one image, select how far to reach left/right/up/down, then rotate the door to reveal different axis-aligned neighborhoods without losing your anchor.

I. Coordinate targeting

Select any image by coordinate and teleport to it

Every image has a stable identity plus a current coordinate under the current arrangement. If you have an image ID (or you click it in any view), you can open its coordinate directly.

1) Choose the anchor image

Pick any image, or paste its ID, then “Go to coordinate.”

2) Land on (x,y,z)

The engine positions you at that exact neighborhood, making it your local origin.

3) Build from the anchor

Everything that follows is computed relative to the selected image.

Anchor Image A selected
A has coordinate: (x,y,z)
User action: Go to A
Result: A becomes local origin for traversal + plane construction
II. Plane construction

Form a plane around the image using offsets

Once you’re standing on an image, you define a “plane window” by choosing distances: how many images to include to the left, right, up, and down. This creates a rectangular neighborhood slice centered on your anchor.

Offsets
Example: Left=3 Right=1 Up=4 Down=2 creates a plane window that reaches unevenly in each direction.
Why uneven matters
You can “lean” the window toward where the cluster is denser, without recentering the anchor image.
What populates the plane
The plane is filled by the nearest images relative to the anchor under the current arrangement, preserving the current geometry of the cluster.
Anchor = A at (x,y,z)
Plane Window = A + offsets
L=3 R=1 U=4 D=2
→ construct an axis-aligned neighborhood plane around A
What this gives you

A “cluster slice” you can screenshot

This is designed so a screenshot isn’t just an aesthetic capture. It’s a navigational artifact: you can use the offsets + dial state to find the same region again.

Anchor stays visible

The selected image remains the constant reference point.

Neighborhood is bounded

Offsets make the plane finite and readable.

Dial makes it dimensional

Rotation swaps which axis you’re “looking along.”

III. Swivel door dial

Turn a dial to rotate axis view

The dial behaves like a swivel: it rotates which axis is emphasized. At one dial position, you’re looking at the top view. Turn the dial and you begin seeing the same neighborhood projected so the Y axis (depth) becomes legible as layers.

Dial = factor
The dial increments by “factors” (example: 1 step per click). Each step corresponds to an axis rotation state.
What changes
Same anchor. Same offsets. Different projection: you see the surrounding images reorganized by the chosen axis view.
Why this matters
It lets you “read depth” in a plane system: you can examine a cluster’s vertical layering without leaving your anchor.
Dial position N → Axis view state N
N=0 → Top view (plane projection)
N=1 → Y-axis emphasis (layered depth)
N=2 → alternate projection (another axis slice)

Anchor + offsets stay constant; only the projection changes.
IV. Save like a scientist

Save the plane snapshot for reconstruction

When you save, you’re not saving only the image. You’re saving the conditions that produced this neighborhood and the plane view you built around it.

Save offsets

Left/Right/Up/Down distances used to construct the plane window.

Save dial rotation

The knob’s position, mapped to the axis projection state.

Save the arrangement

Everything needed to approximate the same cluster again.

What a saved record contains

Full snapshot payload (human-readable)

Dataset
Which database/archive filled the plane (source + version/seed if applicable).
Anchor coordinate
The chosen image coordinate (x,y,z) at save time.
Offsets
Left/Right/Up/Down distances used to create the plane window.
Dial state
Knob rotation value + the axis mapping it implies.
Founding inputs
Seeds used to form the plane: images, tags, tag clusters, and any composites.
Composites
If a point was “image+image+tag cluster,” store each component explicitly.
Heat profile
Heat for every relevant point: tags, clusters, images, and composites.
Tag composition
If a node is a tag cluster, store which tags it contains (and their weights if any).
User note
Optional text you append to the saved item (why it matters, what you found).

Why store this much? If you lose everything, a saved record plus a screenshot can guide you back to the same plane cluster. The anchor coordinate is the “where,” and the offsets + dial state are the “how you looked at it.”
