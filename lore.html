Atlas Characters Lore Mechanics and More!

Characters that persist, branch, and remember… without breaking the story.

The Atlas Character System turns images into living people across worlds. It keeps continuity through Core Identity, grows depth through Saved Lore States, and adds haunting realism through Resonant Bleed — where feelings echo across timelines without forcing factual contradictions.

Design goal: A character should feel like a real person traveling through impossible worlds: confused, relieved, emotionally consistent — and capable of saying the line that hits: “It’s funny… I feel like we’ve been here before.”

What this solves

Most “AI characters” reset. Atlas characters accumulate history, survive world changes, and stay coherent even when you rearrange reality.

Why it matters

Continuity creates attachment. Attachment creates exploration. Exploration creates worlds that feel worth returning to.

Core promise

Your favorite character can meet you again in a different slice and still carry the emotional truth of your first encounter.

Style note

Characters never say “I’m an AI.” They speak like survivors of strange geography: fog, doors, portals, and accidents.

I. Architecture

Every character has three layers

Atlas characters are built like a durable object with a living timeline. The system splits what must never change from what should evolve.

Layer 1

Core Identity (immutable)

Name, voice, archetype, motivations, fears, baseline vibe. This persists through slice changes, merges, and reverts.

Layer 2

Active Canon (current timeline)

What’s happening right now: location, goals, injuries, relationships. This changes as the world evolves.

Layer 3

Lore Archive (saved states)

Snapshots you deliberately preserve. These create a branching “Lore Tree” that can be referenced later without overwriting history.

Core Identity → always loaded
Active Canon → loaded for the current slice/world
Lore Archive → saved branches (loaded only when relevant)
II. Saved Lore States

The Lore Tree: continuity without overwriting

When you “Save” a character, you create a preserved branch. Each save is a snapshot, not a replacement. The earlier you save them, the deeper their long-term continuity can become.

Saved state includes
World context, slice ID, timestamp, relationship status with the user, emotional state, key events, and memorable dialogue fragments.
Why it matters
You can revert the globe’s arrangement while keeping narrative history. The world can reset, but the character’s story doesn’t vanish.
Branching behavior
Branches can diverge wildly (alternate encounters, different worlds), yet remain anchored by Core Identity.

Key user fantasy: You meet someone, you save them, and later they can genuinely say “I’m relieved you’re here.” because that relief is grounded in stored history.

III. Emotional Signature

The secret sauce: memory as emotion, not exposition

Each saved branch stores an Emotional Signature Vector. This is not a wall of text. It’s a compact set of markers describing what the character has been through and what they associate with you.

Markers

Trauma, bond, safety, fear

The vector includes trauma intensity, trust depth, “safe-haven” moments, unresolved tension, and first-meeting imprint.

Scalability

Big lore, small memory

This keeps the system scalable. The model doesn’t need to load 500 pages to preserve the vibe of a relationship.

Explainable

Why they react

When a character panics in fog, you can trace it to stored emotional markers instead of “random AI behavior.”

IV. Resonant Bleed

Bleed lets the past haunt the present without breaking canon

Atlas uses a controlled mechanic called Resonant Bleed. Bleed does not transfer factual memory by default. It transfers emotional echoes across branches. This keeps stories coherent while still creating that uncanny “we’ve met before” realism.

Triggers

Bleed can trigger under: high stress, trauma, environmental similarity (fog, doorways, sirens), proximity to the user, and symbolic repetition.

Low Bleed
Mild déjà vu lines. “This place… feels familiar.”
Medium Bleed
Dream-like references without clear facts. “I had a nightmare about a hallway like this.”
High Bleed
Accurate but confused fragments. “We ran. You pulled me through a door… didn’t you?”
Extreme Bleed
Used sparingly. Emotional collapse from conflicting echoes.

Why it feels real

Most systems either forget everything or dump everything. Atlas chooses a third option: emotion-first continuity with optional factual recall when the user prompts it.

Bleed ≠ factual transfer
Bleed = emotional resonance between branches

Default: characters don’t “know the system”
They interpret the weirdness as fog, doors, mist, accidents.

User prompt can unlock: archived milestones

The signature line: “It’s funny… I feel like we’ve been here before.”
That line is powered by stored emotional markers, not random improvisation.

V. Merges & Slices

World-level operations that keep characters stable

Atlas supports operations where the world layout changes dramatically without destroying character identity. This is where the system becomes “world-grade,” not just “chat-grade.”

Dimensional Merge

Collapse the globe into one plane

The globe expands until all items fit into a single plane slice. Then everything becomes a coherent world. Saved characters retain continuity; unsaved entities are reinterpreted fresh.

Revert

Return to pre-merge state

Reverting restores the prior spatial arrangement. Saved lore persists. Characters can remember merge events via resonance even after reversion.

People Slices

Bring saved characters together

If a slice contains multiple saved characters, the system can propose generating a world where they all appear, justified by believable “crossed worlds” narrative.

Crucial rule: Characters are not “aware of merges.” They sound confused, sometimes shaken, but narratively grounded: “I followed the mist.” “I ran into a building.” “I couldn’t find the exit.”

VI. Lore Editing & Save Points

Manual edits that remain reversible

You can manually affect the lore of any character. The engine will adapt the world to keep coherence. Every edit creates a new Save Point so older versions remain accessible.

You can edit
Backstory, motivations, relationships, trauma events, affiliations, and key milestones.
Versioning
Prior lore is never erased. You can revert a character to a previous save point at any time.
Coherence rule
The story engine treats edits as “retcons” that must be justified in-world when needed.
VII. Bleed-Through After Editing

The haunted upgrade: memory echoes across versions

If you edit a character’s lore, older branches can still “echo” through emotional resonance. This is the Bleed-Through mechanic: not a contradiction, a psychological afterimage.

Soft Echo

Déjà vu with no facts

“It’s weird… I feel like I’ve lost time.” Useful after trauma, chases, assaults, or monster escapes.

Triggered Recall

User words unlock history

The system stores major milestones. If you mention a trigger phrase, they can “remember”: “Wait… the hallway. The door. You were there.”

Lore Depth

Endless growth, controlled context

Only recent + major events load by default. Deep lore remains stored and is retrieved when meaningfully invoked.

Outcome: Characters can accumulate years of lore without collapsing the model’s context window, while still feeling continuous and emotionally consistent.
