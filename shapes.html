The Shape of Our Vision

Pick 4 images. Atlas forms the planes between them… and a 3D box appears.

Shapes are the fastest way to see intersectionality at a glance. You choose a handful of anchors inside a saturated cluster, and Atlas skins the faces with the images already living there. It’s a saved object you can return to, rotate mentally, and annotate like a real artifact.

Why it works

Because the coordinate space is already populated. The shape is just a clean slice of a dense neighborhood.

Best conditions

Ultra-saturated regions: lots of nearby images means the planes can “tile” naturally without gaps.

Not gimmicky

It’s just a saved, bounded view of meaning. A box is easier to understand than a thousand points.

What you get

A shape you can name, revisit, share, and annotate. The anchors and constraints are stored automatically.

1. The core move

How shapes are made (super easy)

Step 1

Pick 4 images in one cluster

Select four anchors that already live near each other. Atlas infers the quadrilateral ordering and uses them as corners.

Step 2

Planes form between the images

Atlas spans planes between the selected anchors, creating a face that represents “what exists between these meanings.”

Step 3

The face auto-populates

Because images are already placed in coordinate space, the face is filled by adjacent images that sit on that region.

What is a “face” here?
A face is a bounded plane inside a dense neighborhood. It’s a readable surface made from what already exists nearby.
Why 4?
Four anchors are enough to define a quadrilateral surface that can be populated without requiring complicated user input.
2. Depth

Two ways to give the face a 3D box

Option A

Distance Up / Down

You choose a depth amount above and below the face. Atlas forms the top and bottom planes at those offsets.

Option B (optional)

Pick an image above + an image below

Instead of a number, you select a top anchor and a bottom anchor. Those define the top and bottom planes directly.

Result

A quadrilateral 3D box

The face becomes a box with multiple populated planes. The shape “fills” best when the cluster is ultra-saturated.

4 image anchors → define the main face
Depth Option A: distance up/down → top plane + bottom plane
Depth Option B: image above + image below → top plane + bottom plane

Then: side planes connect → quadrilateral 3D box
Faces populate from nearby images because the space is already dense.
3. Saving

Save the shape, annotate the images, keep the constraints

Automatic

Anchors + constraints are stored

The shape stores which images formed it, the face definition, and the depth definition (numeric or top/bottom images).

Notes

Text per image or for the whole shape

Add a caption to any image on a face, or attach a single explanation describing what this shape represents.

Purpose

Fast recall of intersectionality

Shapes are saved artifacts. They exist so you can see a multi-meaning intersection instantly, without re-finding it.

What gets saved
The 4 face images, the depth method (up/down or top/bottom images), and the rules that produced the populated faces.
Why density matters
Ultra-saturated clusters have enough adjacent images to populate the planes cleanly. Sparse regions produce thin faces.
What you can do with it
Save it as a named object, revisit it anytime, and use it as a quick “at-a-glance” map of a dense intersection.
