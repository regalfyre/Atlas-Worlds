Premise

The industry has chased graphics for decades. We are chasing something more fundamental: meaning under pressure.

We propose a new category: persistent semantic worlds that don’t just look like places, but behave like places with consequence, memory, and identity.

Thesis: Text becomes ontology. Ontology becomes structure. Structure becomes space. Space produces interaction. Interaction becomes memory. Memory becomes continuity.

The Problem

AI talks, but it doesn’t inhabit space

Chat systems can roleplay endlessly, but the experience is flat. There is no geography, no distance, no risk, no traversal, no pressure. The world doesn’t push back.

Missing: embodied consequence

Games simulate space, but they don’t understand meaning

Most worlds are scripted or procedurally noisy. NPC dialogue rarely affects the world, and “memories” rarely become durable, recallable, emotionally anchored artifacts.

Missing: semantic continuity

The gap is clear: one side has language without space, the other has space without meaning. From Words to Worlds is the bridge.

Core Innovation

This is not “procedural generation.” It is meaning-first world synthesis. The LLM does not output a story as a wall of text. It outputs a World Specification: structured, constrained, and engine-compilable.

Key separation: Unity governs physics and movement. The LLM governs meaning, dialogue, continuity, and the semantic glue that makes it feel alive.

The Pipeline (Beginning to End)

Stage 1: Curated Genesis Creator-controlled

We can generate the initial character images and starting lore canon ourselves. The goal is cohesion and identity from day one.

  • Character images (canonical look, consistent style)
  • Initial lore (up to ~2 pages per character)
  • Motivations, relationships, tensions, triggers
  • Signature items and social roles
Strong canon prevents “generic NPC drift.” The LLM extends authored DNA instead of inventing chaos.

Stage 2: Automatic Meaning Extraction Vision layer

If an uploaded image lacks a text pairing, the system creates one automatically. Every visual asset becomes semantically active world fuel.

  • Vision model generates caption + tags + emotional tone
  • Produces a structured summary suitable for world-building
  • Feeds the LLM as authoritative “what this asset means” input
Input Image → Vision Caption / Tags → Structured Summary (objects, tone, implications) → LLM World-Building Input
Guarantee: nothing visual is inert. Everything has meaning, even if the user provides no text.

Stage 3: Structured World Generation LLM

The LLM composes a coherent world spec: locations, links, placements, threats, behaviors, and hooks. It fills meaning into a constrained skeleton, not a freeform hallucination.

  • Locations tagged SAFE / DANGER / INSTITUTION / TRANSIT
  • Traversal links (door, street, trail)
  • Item placement + affordances
  • Character placement + roles + dialogue style
  • Threat vectors and pressure zones
Anti-clustering rule set: Characters are spread throughout the map with 0–2 per location (pairs max). Several locations begin empty to encourage exploration.

Stage 4: Unity Compilation Engine

Unity reads the JSON world spec and instantiates the world using templates and prefabs. v1 can be primitive: box rooms, capsule NPCs, cube items. Traversal is the proof.

  • Locations → room/zone templates
  • Links → trigger volumes (doors/exits)
  • Characters → NPC prefabs (NavMesh agents)
  • Items → interactable props
Outcome: a playable world you can walk immediately, even with zero custom art.

The Experience Loop

The world runs on two forces: Exposure (open space) and Shelter (door zones). That tension generates urgency and attachment.

Open Space (Exposure) → Threat / Chase / Health Pressure → Retreat / Hide / Regroup → Door Zone (Shelter) → Deep Interaction + Milestones → Memory + Artifact → Back to Exposure (with continuity)

Behind-the-Door Zones (The Heart)

Door Zones are not “interiors.” They are semantic state changes. Crossing the threshold changes how the world behaves and how meaning is generated.

What changes at the threshold

  • Threat pressure drops (danger does not follow inside)
  • Dialogue budget expands (depth replaces reactivity)
  • Lore recall increases (key memories become active context)
  • Milestone logging activates (meaning becomes durable)
  • Action set expands (talk deep, ritual, vow, comfort, plan)

Why Door Zones matter

  • Open space creates urgency, but not attachment.
  • Door Zones create relief, and relief enables bonding.
  • Bonding creates memories that change future dialogue and behavior.
  • Door Zones are where the world grows roots instead of noise.
Door Zones are memory forges: They turn lived moments into structured milestones the world can reference.

Door Zone Types

  • Safe Shelter: hideouts, homes, back rooms (heal, comfort, bond)
  • Institutional Zones: courts, offices, ceremonies (formal consequences)
  • Intimate Zones: private spaces (confession, vulnerability, relational depth)
Institutions aren’t set dressing. They are semantic state-change environments that formalize decisions.

Open Space vs Door Zone Mode

Open Space Mode: - movement priority - threat detection high - dialogue short/reactive - low lore recall - few verbs (run, hide, inspect) Door Zone Mode: - movement minimal - threat disabled/reduced - dialogue deep - lore recall active - milestones enabled - expanded verbs (vow, ritual, comfort, plan)

Memory & Commemorative Artifacts

Milestones become structured memory

When something meaningful happens in a Door Zone, it is recorded as a structured event. This is how the world earns continuity without exploding into infinite token storage.

{ "event_type": "MILESTONE", "participants": ["player", "ch_4"], "location": "loc_back_room", "emotional_tags": ["trust", "relief", "bonding"], "impact_score": 8, "summary": "She said it was the best day of her life." }
Key: We don’t delete memories. We compact them. The world stays coherent without becoming expensive chaos.

Door Zones generate commemorative images

The Door Zone is also a memory studio. When a milestone forms, the system can render it: your character plus the other character(s), in the scene you just lived.

  • Milestone → image prompt (tone + environment + actions)
  • Character consistency from canonical images
  • Generated image saved as a lore artifact
  • Paired with a concise caption for future recall
Milestone (structured) → Image Prompt (scene + emotion) → Generated Commemorative Image → Caption/Text Pairing → Stored as Lore Artifact
The memory becomes visual proof. Later dialogue gains weight because the artifact exists.

Elastic Memory (Compaction, Not Hostage)

Continuity is preserved through a fair memory architecture: detailed recent events and user-selected “key memories” stay vivid, while everything else is compacted into summaries that remain expandable.

  • Recent events: stored in richer detail
  • User-selected Top Memories: always available in depth
  • Everything else: compact summaries + tags
  • No deletion: memory returns when the context budget returns
The promise is preservation. The mechanism is compaction. The result is coherence without runaway cost.

Semantic-Games v1 (Working Prototype ASAP)

v1 is intentionally minimal: a proof that the world compiles, traversal works, characters move, threats pressure movement, and Door Zones produce depth and memories.

Unity Components

  • WorldJSONLoader (reads world spec)
  • RoomBuilder (templates per location)
  • LinkSpawner (door/path triggers)
  • EntitySpawner (NPCs + items)
  • TraversalManager (location transitions)
  • NPCController (follow/flee/chase)
  • DoorZoneController (mode switch)
  • EventLogger (milestone logs)

v1 Proof Checklist

  • JSON loads into a traversable world
  • Characters distributed (pairs max per location)
  • Companions follow and flee
  • Threat patrols, detects, chases
  • Door Zone triggers deep interaction mode
  • Milestone log writes correctly
  • Milestone → commemorative image + caption
  • Lore updates reference milestones later
v1 goal: prove “words → world” with the exposure/shelter loop and a memory system that the world can call back later.
